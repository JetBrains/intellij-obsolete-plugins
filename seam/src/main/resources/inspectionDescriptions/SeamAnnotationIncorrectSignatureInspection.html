Reports Seam annotations that appear on classes or methods with invalid signatures or fields with invalid type.

  • @Name is invalid on interfaces and abstract classes. Requires a no-argument constructor in the annotated class.

  • @In must appear either on a field or on a setter method, such as void setUser(User user).

    • Void return type
    • One argument
    • Non-static
    • Name must start with set

  • @Out must appear either on a field or on a getter method, such as User getUser().

    • Non-void return type
    • No arguments
    • Non-static
    • Name must start with get, or optionally is if the return type is boolean

  • @DataModel must appear either on a field or on a getter method (same as the @Out annotation). The field type or return type must be one of the following:

    • java.util.List
    • java.util.Set
    • java.util.Map

  • @DataModelSelection must appear on setter methods.

  • @DataModelSelectionIndex must appear on setter methods.

  • @Factory and @Unwrap must appear on a method with the following signature:

    • Non-void return type
    • Either no parameters or a single org.jboss.seam.Component parameter (or its super type)
    • Non-static

  • @Create must appear on a method with the following signature:

    • Either no parameters or a single org.jboss.seam.Component parameter (or its super type)
    • Non-static

  • @Destroy must appear on a method with the following signature:

    • Void return type
    • No parameters
    • Non-static
