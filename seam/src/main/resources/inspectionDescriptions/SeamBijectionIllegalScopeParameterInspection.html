Reports the scope parameters in @In and @Out annotations for cases when Seam always uses the scope defined by the component itself.

  • When specifying @In(create=true), the context variable must belong to a declared Seam component. Else, Seam would not know which class to instantiate. In other words, for create=true you cannot inject values that are not Seam components. Seam always uses the scope defined in the component declaration (the default scope for the component type or the scope defined in @Scope or @Role).
  • When you eject a value of a declared Seam component with @Out, the scope is always taken from that component. You can use @Out(scope=...) only when you are not ejecting a Seam component.
